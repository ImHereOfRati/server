| 시도 | 실패 원인 |
|------|---------|
| Kotlin 보간법을 MockMvc URL에 이스케이프 (\${id}\)로 작성 | 이스케이프되어 문자열에 그대로 입력됨에 따라 IllegalArgumentException 발생 |
| 토큰 값을 이스케이프 (\\\)로 작성 | "Bearer \" 문자열이 그대로 삽입되어 401 Unauthorized 발생 |
| DTO 필드의 제약조건(\@Length(min=10)\)을 충족하지 못한 짧은 문자열 전송 | 비즈니스 로직(409/403/404)에 도달하기 전 Validation에서 튕겨 400 Bad Request 발생 |
| FriendException 클래스의 HTTP Status 매핑이 기획/코드와 다름 | 예상한 403 대신 400(수신자 불일치) 또는 422(타겟 차단)로 반환되어 테스트 실패 발생 |
| RestDocs 문서화 시 \elaxedResponseFields\를 사용했으나 응답에서 실제로 누락(null)되는 필드 발생 | Jackson 직렬화 시 \NON_NULL\ 설정으로 인해 \data.id\ 등 null 데이터가 누락되어 SnippetException 발생 |
| \elaxedResponseFields\ 문서화 중 응답 모델에 추가된 필드(객체)를 문서화하지 않음 | 내부 객체(\data.restrictor\, \data.restricted\ 등)를 명시하지 않아 문서화 실패 발생 |

### Fixes Applied
- \\$\ 이스케이프를 제거하여 Kotlin 템플릿 엔진이 정상 동작하도록 수정
- DTO \@Valid\ 제약 조건에 맞게 테스트 데이터의 문자열 길이를 모두 10자 이상으로 수정
- 도메인 Exception에 매핑된 HTTP Status(\isUnprocessableEntity()\, \isBadRequest()\)에 맞춰 Assert 구문 변경
- RestDocs의 필드 명세 시 \.optional()\을 추가하고, \subsectionWithPath()\를 사용하여 중첩 객체 문서화를 완료하여 SnippetException 해결
